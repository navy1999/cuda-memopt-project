# CUDA Matrix Multiplication Optimizations

**Run all commands below from this directory (`cuda/`).** The repo root is the Next.js frontend; this folder holds the CUDA project.

This project combines hand-tuned CUDA matrix multiplication kernels with a compiler-driven LLVM tiling pass and an autotuner. Follow the instructions below to build and run both the CUDA and LLVM components.

## Directory structure

```
cuda-memopt-project/
├── src/                    # CUDA source files
│   ├── matmul_naive.cu
│   ├── matmul_tiled.cu

│   ├── matmul_tiled_vec_unroll.cu
│   ├── matmul_tuned.cu
│   └── cuda_utils.h        # Shared CUDA error-checking helpers
├── llvm-pass/              # LLVM loop-tiling pass
│   ├── include/LoopTiling.h
│   ├── LoopTiling.cpp
│   └── CMakeLists.txt
├── scripts/
│   ├── benchmarking.py     # CUDA benchmarking and plotting
│   ├── autotune.py         # LLVM autotuner
│   └── plot_autotune.py    # Plot autotune results
├── build/                  # Compiled executables (and optional LLVM plugin build)
├── results/                # Generated CSVs and plots
└── report/                 # Project report (e.g. report_compiler.pdf)
```

## Prerequisites

- **NVIDIA GPU** with CUDA support and **CUDA Toolkit** (`nvcc` in PATH)
- **Visual Studio Build Tools** (Windows) or **GCC** (Linux)
- **Python 3.x** with `numpy` and `matplotlib`
- **CMake** ≥ 3.10 (≥ 3.20 for the LLVM pass)
- **LLVM** toolchain (`clang++`, `opt`, `llvm-config`) on PATH for the autotuner

## Building and running CUDA kernels

From the project root:

```bash
mkdir -p build
nvcc -O3 -o build/matmul_naive.exe     src/matmul_naive.cu
nvcc -O3 -o build/matmul_tiled.exe     src/matmul_tiled.cu
nvcc -O3 -o build/matmul_tiled_vec.exe src/matmul_tiled_vec.cu
nvcc -O3 -o build/matmul_tiled_vec_unroll.exe src/matmul_tiled_vec_unroll.cu
nvcc -O3 -o build/matmul_tuned.exe     src/matmul_tuned.cu
```

On Linux/macOS, omit the `.exe` suffix. To use **CMake** instead (configurable CUDA arch):

```bash
cmake -S . -B build -DCMAKE_CUDA_ARCHITECTURES=61   # or 75, 86, etc. for your GPU
cmake --build build --config Release
```

Run a single test:

```bash
build/matmul_naive.exe 512
build/matmul_tiled.exe 512
```

**Note:** Vectorized kernels (`matmul_tiled_vec`, `matmul_tiled_vec_unroll`) require `N` divisible by 4. The tuned kernel requires `N` divisible by 32.

## Benchmarking CUDA kernels

From the project root:

```bash
python scripts/benchmarking.py
```

This will compile all CUDA variants, run them over sizes 256, 512, 1024, 2048 (5 trials each), and produce:

- **results/benchmark.csv**
- **results/comparison.png**

Options:

- `--no-compile` — use existing executables in `build/` (skip compile)
- `--sizes 256,512,1024` — matrix dimensions to test
- `--trials 10` — number of trials per size
- `--versions naive,tiled` — only run these variants
- `--plot-only` — only regenerate the plot from existing `results/benchmark.csv`

## Building the LLVM loop-tiling pass

Install a full LLVM distribution so that `opt`, `clang++`, and `llvm-config` are on your PATH.

From the project root:

```bash
cd llvm-pass
mkdir -p build && cd build
cmake .. -DLLVM_DIR="C:/Program Files/LLVM/lib/cmake/llvm"   # Windows: adjust path
# On Linux/macOS, use your LLVM install path, e.g. -DLLVM_DIR=/usr/lib/cmake/llvm
cmake --build . --config Release
```

The plugin **LoopTilingPass.dll** (Windows) or **LoopTilingPass.so** (Linux/macOS) will be in `llvm-pass/build/Release/`.

## Running the autotuner

From the project root (after building the LLVM pass):

```bash
python scripts/autotune.py
```

This will lower `matmul_naive.cu` to LLVM bitcode, apply the tiling pass at tile sizes 8, 16, 32, 64, recompile with `nvcc`, and write **results/autotune.csv**. Options:

- `--pass-plugin /path/to/LoopTilingPass.so` — path to the plugin (default: auto-detect under `llvm-pass/build/`)
- `--sizes 8,16,32` — tile sizes to try
- `-N 2048` — matrix dimension (default 1024)
- `--cuda-arch sm_75` — GPU architecture for clang (default sm_61)

Plot autotune results:

```bash
python scripts/plot_autotune.py
```

Output: **results/autotune.png**.

## Troubleshooting

- **opt / llc not found** — Install the full LLVM package or add `LLVM/bin` to your PATH. On Windows, close and reopen the terminal (or run `refreshenv` if using Chocolatey).
- **Wrong GPU architecture** — Pass `-DCMAKE_CUDA_ARCHITECTURES=86` (or your GPU’s compute capability) when configuring CMake, or set `--cuda-arch sm_86` in `autotune.py`.
- **LoopTilingPass not found** — Build the pass first (see above). Use `--pass-plugin` to point to the correct `.dll` or `.so` path.

You now have a complete, reproducible workflow for both manual CUDA optimizations and compiler-driven tiling with LLVM.
